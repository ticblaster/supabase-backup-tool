Corrija um bug na ferramenta supabase-backup-tool.

Erro observado:
Ao rodar npm run backup:db -- --project geral-homologacao, o comando de roles foi gerado assim:

supabase db dump --db-url ... -f roles.sql --role-only -x storage.buckets_vectors -x storage.vector_indexes

A Supabase CLI falhou com:
required flag(s) "data-only" not set

Causa:
A ferramenta está aplicando db.excludeTables em todos os dumps, inclusive roles.sql. Isso está errado.

Correção obrigatória:
1. Em src/backup/backup-db.ts, aplicar excludeTables somente no comando de data.sql.
2. O comando roles.sql deve ser:
   supabase db dump --db-url "$DB_URL" -f roles.sql --role-only
3. O comando schema.sql deve ser:
   supabase db dump --db-url "$DB_URL" -f schema.sql
4. O comando data.sql deve ser:
   supabase db dump --db-url "$DB_URL" -f data.sql --use-copy --data-only -x storage.buckets_vectors -x storage.vector_indexes
5. Se não houver excludeTables, data.sql deve rodar sem flags -x.
6. Atualizar logs para mostrar claramente:
   [db] Gerando roles.sql...
   [db] Gerando schema.sql...
   [db] Gerando data.sql...
7. Adicionar teste simples ou validação interna que garanta:
   - --role-only nunca recebe -x
   - schema dump nunca recebe -x
   - -x só aparece junto com --data-only
8. Manter npm run typecheck e npm run build verdes.
