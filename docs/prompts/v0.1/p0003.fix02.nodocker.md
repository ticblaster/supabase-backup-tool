Ajuste a ferramenta supabase-backup-tool para não depender obrigatoriamente de Docker/Supabase CLI no backup do banco.

Problema observado:
Ao rodar:

npm run backup:db -- --project geral-homologacao

A ferramenta executou:

supabase db dump --db-url ... -f roles.sql --role-only

E falhou com:

failed to run docker. Docker Desktop is a prerequisite for local development.

Contexto:
No ambiente Windows atual, psql, pg_dump e rclone já estão instalados e funcionando. Portanto, a ferramenta deve conseguir fazer backup do banco usando ferramentas nativas do PostgreSQL, sem Docker.

Objetivo:
Adicionar suporte a dois engines de backup de banco:

1. "supabase-cli"
2. "pgdump"

O padrão recomendado deve ser "pgdump".

Configuração esperada no backup.config.json:

{
  "supabaseCli": "npx supabase",
  "defaultProject": "geral-homologacao",
  "backupRoot": "./backups",
  "projects": {
    "geral-homologacao": {
      "projectRef": "izzitgselrzkxsqglcja",
      "dbUrlEnv": "GERAL_HOMOLOGACAO_DB_URL",
      "db": {
        "engine": "pgdump",
        "excludeTables": [
          "storage.buckets_vectors",
          "storage.vector_indexes"
        ]
      },
      "storage": {
        "mode": "rclone",
        "remote": "supabase-geral",
        "buckets": ["trip-receipts"],
        "rcloneFlags": ["--s3-list-version", "2"]
      }
    }
  }
}

Requisitos para engine "pgdump":

1. Gerar roles.sql usando pg_dumpall, se disponível:

pg_dumpall --roles-only --no-role-passwords --dbname "$DB_URL" --file roles.sql

2. Se pg_dumpall não estiver disponível ou falhar por permissão, não abortar o backup inteiro.
   Criar roles.sql com comentário de aviso, por exemplo:

-- roles.sql não gerado automaticamente.
-- Motivo: pg_dumpall indisponível ou permissão insuficiente.
-- Para projetos Supabase, roles gerenciadas podem exigir recriação manual/validação no restore.

3. Gerar schema.sql com pg_dump:

pg_dump --dbname "$DB_URL" --schema-only --no-owner --no-acl --file schema.sql

4. Gerar data.sql com pg_dump:

pg_dump --dbname "$DB_URL" --data-only --no-owner --no-acl --file data.sql

5. Aplicar excludeTables somente no data.sql usando flags do pg_dump:

--exclude-table=storage.buckets_vectors
--exclude-table=storage.vector_indexes

6. Nunca aplicar excludeTables em roles.sql nem schema.sql.
